# Teacher Model 1：YOLOv8x-HBB 原图监督基线 v1.0.0

## 1. 版本定位

`v1.0.0` 是本项目第一个可运行、可复现的监督学习基线版本。

本版本以 YOLOv8x 作为 Teacher 模型主干，直接使用转换后的 DOTA 五分类子集进行训练，未引入 SAHI 切片增强，也未使用半监督学习或伪标签机制。

该版本的主要目的不是追求最终精度，而是验证完整训练流程的可行性，并作为后续 Teacher 模型迭代的对照基线。

## 2. 实验设置

| 项目 | 设置 |
|---|---|
| 模型 | YOLOv8x |
| 数据集 | DOTA 五分类子集 |
| 类别 | plane, ship, vehicle, harbor, storage_tank |
| 标签格式 | DOTA OBB → YOLO HBB |
| 输入尺寸 | 1024 |
| Batch Size | 4 |
| Epochs | 100 |
| 训练时间 | 约 1.126 小时 |
| 验证集图像数 | 282 |
| 验证集目标实例数 | 42,844 |

| mAP50 | 0.475 |

## 5. 结果分析

第一版基线模型成功验证了 YOLOv8x 在 DOTA 五分类子集上的训练流程可以完整跑通，说明数据转换、标签格式、训练配置和验证流程整体可行。

从整体指标来看，模型取得了 `0.475` 的 mAP50 和 `0.258` 的 mAP50-95，说明原图监督训练具备一定检测能力，但距离高质量 Teacher 模型仍有明显差距。

各类别表现差异较大：

- `plane` 类别表现最好，Precision、Recall 和 mAP 均较高，说明尺寸相对较大、形状特征明显的目标更容易被模型学习。
- `ship` 和 `storage_tank` 取得了中等水平的检测效果，说明模型能够学习到一定的类别特征，但召回率仍然存在提升空间。
- `vehicle` 类别虽然实例数量最多，达到 33,214 个，但检测效果最差，Recall 仅为 0.201，mAP50 仅为 0.185，是当前版本的主要瓶颈。
- `harbor` 类别表现同样较弱，可能与样本数量较少、目标边界不清晰、背景结构复杂有关。

其中，`vehicle` 类别的结果尤其关键。它不是因为样本数量不足而表现差，而是因为遥感图像中的 vehicle 往往尺寸极小、分布密集，原始大图直接缩放到 `1024×1024` 后，小目标特征被严重压缩，导致模型难以学习到有效特征。

因此，v1.0.0 暴露出的核心问题是：

```text
原图监督训练可以跑通，
但遥感小目标在缩放后特征丢失严重，
尤其影响 vehicle 等密集小目标类别的检测效果。
